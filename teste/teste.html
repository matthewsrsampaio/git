Aprendendo GIT

Está organizado em Ações ou algo específico como Repositório, por exemplo.

Códigos Usados

Adições

Adicionar o nome de usuário GLOBAL: git config --global user.name "SeuNome"

Adicionar o email GLOBAL: git config --global user.email "seuemail@seuemail.com"

Adicionar o nome de usuário LOCAL: git config user.name "SeuNome"

Adicionar o email LOCAL: git config user.email "seuemail@seuemail.com"

Deletar

Deletar o repositório local: rm -rf .git

Verificações

Verificar os registros de commit: git log

Verificar os 2 ultimos registros de commit: git log -2

Verificar somente a mensagem dos 2 ultimos registros de commit: git log --oneline -2

Verificar os registros de commit com gráfico histórico: git log --graph

Verificar o log a partir do nome da branch: git log nomeDaBranch

Verificar os registros de commit pelo nome do autor: git log --author="nomeDoAutor"

Verificar os registro resumidos: git shortlog

Verificar o registro das ações feitas dentro do repositório: git reflog

Verificar informações sobre o remote: git remote show origin

Verificar o repositório remoto: git remote -v

Verificar informações globais do GIT: git config --global --list

Verificar informações locai do GIT: git config --list

Verificar o status do repositório: git status

Verificar se ainda existe arquivos e pastas escondidas: ls -lah

Commit

Tirar um arquivo do STAGE, mas mantendo a configuração feita: git reset nomeDoArquivo

Manter um arquivo no STAGE, mas apagando as modificações feitas: git restore nomeDoArquivo

Ver a diferença dos arquivos em STAGE para os que não estão: git diff

Commitar os arquivos em STAGE: git commit -m "mensagem explanatória sobre o que commir realizado"

Commitar os arquivos em STAGE adicionando algo que esteja fora do STAGE: git commit -a -m "mensagem explanatória sobre o que commir realizado"

Fazer um ajuste de última hora em um arquivo ja commitado, digite: git add . e depois digita: git commit --amend -m "mensagem explanatória"

Diferença do último commit pro STAGED: git diff --staged (Lembrar de adicionar os arquivos ao STAGE antes de enviar o comando.)

Repositório

Iniciar um repositório: git init

Clonar um repositório: git clone endereçoHTMLdoRepositório.git

Adicionar um repositório remoto: git remote add origin endereçoDpRepositorioRemoto.git

Remover um repositório remoto: git remote remove origin endereçoDpRepositorioRemoto.git

Mudar o endereço de repositório remoto: git remote set-url origin nomeNovoEndereco.git

Branch

Criar uma branch: git branch nomeDaBranch

Criar e mudar automaticamente para a nova branch: git checkout -b nomeDaBranch

Deletar uma branch: git branch -D nomeDaBranch

Renomear uma branch: git branch -m NovoNomeDaBranch

Ver a diferença entre duas branches: git diff nomeDaBranch nommeDaOutraBranch

Push

Enviar os commits: git push -u origin nomeDaBranch

Pull

Trazer todas as atualizações da branch remota para meu repositório local: git pull origin nomeDaBranch

Merge

**Para mergear(mesclar) uma branch com a "main" deve-se fazer primeiro o checkout para a main(git checkout main) e depois fazer o merge(git merge nomeDaBranch).

Executar um merge: git merge nomeDaBranch

Abortar um merge(caso de conflito): git merge --abort

Continuar merge. Em caso de não abortar um merge e com todos os conflitos resolvidos mergear: git merge --continue
